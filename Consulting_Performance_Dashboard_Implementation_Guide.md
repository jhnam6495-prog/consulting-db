# 컨설팅 수행 실적 관리 워크플로우 대시보드 구현 가이드

## 개요
이 문서는 컨설팅 수행 실적 관리 워크플로우 대시보드를 코드로 구현할 때 단계별로 진행할 수 있는 가이드입니다. 각 단계는 설계, 백엔드, 프론트엔드, 모바일/PC 대응, 승인 워크플로우, 분석/대시보드, 테스트/배포로 구성됩니다.

---

## 1단계: 프로젝트 초기화

### 목표
- 전체 프로젝트 구조 수립
- 프론트엔드와 백엔드 초기 템플릿 생성
- 기술 스택 명세 확정

### 작업
1. 기술 스택 선택
   - 프론트엔드: React + TypeScript + Vite
   - 백엔드: Node.js + Express + TypeScript
   - DB: PostgreSQL + Prisma

2. 초기 폴더 구조 작성
   - `/frontend`
   - `/backend`
   - `/prisma`
   - `package.json`, `tsconfig.json`

3. 클로드 코드 요청 예시
   - "React + TypeScript + Vite / Node.js + Express + TypeScript / PostgreSQL 프로젝트 구조를 생성해줘."
   - "컨설팅 실적 대시보드 앱의 기본 폴더 구조와 초기 파일들을 만들어줘."

### 검증
- `npm install`이 정상 실행되는지 확인
- 프론트엔드와 백엔드 모두 `npm run dev` 또는 `npm start`가 시작되는지 확인

---

## 2단계: 데이터 모델 및 DB

### 목표
- 핵심 엔티티 정의
- DB 스키마 구현
- 마이그레이션 준비

### 핵심 모델
- `Project` / `Performance`
- `Client`
- `User`
- `Team`
- `Approval`

### 클로드 코드 요청 예시
- "PostgreSQL과 Prisma를 사용해 컨설팅 실적 모델 스키마를 만들어줘."
- "프로젝트, 고객사, 실적, 승인, 사용자 엔티티를 포함한 Prisma schema 코드를 작성해줘."

### 검증
- `npx prisma migrate dev --name init` 실행
- 생성된 테이블을 DB에서 확인

---

## 3단계: 백엔드 API 구현

### 목표
- CRUD API 구현
- 승인/반려 흐름 구현
- KPI 및 대시보드 API 추가

### 핵심 API
- `POST /api/performance`
- `GET /api/performance`
- `PUT /api/performance/:id`
- `POST /api/performance/:id/approve`
- `POST /api/performance/:id/reject`
- `GET /api/dashboard/summary`

### 클로드 코드 요청 예시
- "컨설팅 실적 등록, 조회, 수정, 승인, 반려 API를 Express + TypeScript로 작성해줘."
- "승인 대기 목록과 KPI 통계 API를 포함한 라우터를 만들어줘."

### 검증
- Postman 또는 curl로 API 호출
- `GET /api/performance`가 빈 배열 또는 정상 데이터를 반환하는지 확인
- 승인/반려 요청이 상태를 변경하는지 확인

---

## 4단계: 프론트엔드 기본 화면

### 목표
- 주요 화면 UI 구성
- API 연동
- 기본 반응형 레이아웃 구현

### 핵심 화면
- 대시보드 홈
- 실적 목록 / 검색
- 실적 등록 / 수정
- 승인 대기 / 승인 처리
- KPI 요약 카드

### 클로드 코드 요청 예시
- "React + TypeScript로 실적 목록, 실적 등록 폼, 승인 대시보드 화면 컴포넌트를 만들어줘."
- "반응형 레이아웃을 고려해서 모바일 카드뷰와 PC 테이블뷰를 구성해줘."

### 검증
- `npm run dev` 실행
- 브라우저에서 화면 렌더링 확인
- API 데이터가 정상 표시되는지 확인

---

## 5단계: 모바일 / PC 반응형 대응

### 목표
- 한 코드베이스에서 모바일과 PC 모두 지원
- 화면별 레이아웃 최적화
- 사용자 경험 일관성 유지

### 작업
1. 모바일 카드형 리스트 작성
2. PC 테이블형 상세 리스트 작성
3. 필터 패널 및 검색 UI 반응형 처리
4. 주요 버튼을 손가락 터치에 최적화

### 클로드 코드 요청 예시
- "mobile-first 스타일로 모바일 카드형 리스트와 PC 테이블형 리스트를 모두 지원하도록 CSS/스타일을 작성해줘."
- "React 컴포넌트에 반응형 그리드, 토글 가능한 필터 패널을 추가해줘."

### 검증
- 브라우저 개발자 도구로 모바일/데스크톱 화면 확인
- 주요 화면이 breakpoint에서 적절히 변경되는지 확인

---

## 6단계: 승인 워크플로우 완성

### 목표
- 실적 승인/반려 흐름 완성
- 승인 이력 저장 및 표시
- 알림 피드백 제공

### 작업
1. 승인 요청 버튼 구현
2. 반려 사유 입력 모달 구현
3. 승인/반려 API 호출 및 상태 표시
4. 승인 이력 업로드 및 확인 화면 제작

### 클로드 코드 요청 예시
- "컨설턴트가 실적 등록 후 팀장에서 승인/반려하는 흐름을 완성해줘."
- "승인 요청, 반려 사유 입력 모달, 승인 이력 표시 기능을 구현해줘."

### 검증
- 실적 등록 후 승인 요청 시 상태 변경 확인
- 반려 시 사유가 저장되고 화면에 표시되는지 확인

---

## 7단계: 분석 / 대시보드 추가

### 목표
- KPI 및 실적 통계 시각화
- 대시보드 요약 제공
- 필터 기반 분석 지원

### 핵심 데이터
- 기간별 매출 추이
- 서비스 유형별 실적 비율
- 담당자 / 고객사별 집계
- 목표 대비 달성률

### 클로드 코드 요청 예시
- "기간별 매출 추이 그래프와 서비스별 실적 비율 차트를 React로 생성해줘."
- "목표 달성률, 신규 수주 건수, 완료율을 보여주는 KPI 카드 컴포넌트를 작성해줘."

### 검증
- 차트가 정상 렌더링되는지 확인
- 필터 선택 시 데이터가 갱신되는지 확인

## 8단계: 테스트 및 배포 준비

### 목표
- 자동화 테스트 도입
- Docker 및 배포 설정 준비
- CI/CD 흐름 고도화

### 작업
1. 백엔드 Jest 테스트 추가
2. 프론트엔드 React Testing Library 테스트 추가
3. `Dockerfile`, `docker-compose.yml` 작성
4. 기본 배포 스크립트 작성

### 클로드 코드 요청 예시
- "백엔드 API에 대한 간단한 Jest 테스트를 작성해줘."
- "프론트엔드 주요 컴포넌트에 대한 React Testing Library 테스트를 작성해줘."
- "Dockerfile과 docker-compose.yml을 만들어줘."

### 검증
- `npm test` 실행
- `docker build`가 성공하는지 확인
